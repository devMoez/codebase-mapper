# 🌌 Codebase Mapper: Central Nervous System (CNS)

Codebase Mapper (CNS) is a high-performance, structural indexing engine designed to be the "Central Nervous System" of an AI-driven development environment. It solves the context bottleneck by providing a pre-built, always-up-to-date knowledge graph of your project, allowing AI agents to navigate large codebases with surgical precision and minimal token waste.

## 🚀 Key Features

- **🔄 Incremental Watch Mode**: Real-time monitoring (`--watch`) that updates the map in microseconds as you code.
- **🔍 Syntax-Aware Deep Scanning**: Extracts function signatures, class hierarchies, interfaces, and exported symbols using `ts-morph`.
- **🔗 Advanced Dependency Analysis**: Resolves transitive dependencies and automatically detects "Dead Code" (unused files).
- **🤖 AI-Ready Context Packets**: Generates structured summaries (`.codemap/summary.json`) and annotated READMEs designed for AI ingestion.
- **📝 Self-Documentation Stubs**: Automatically writes `// Purpose:` headers back into files based on extracted symbol metadata.
- **🛡️ Intelligent Ignore**: respects `.gitignore`, `.dockerignore`, and `.codemapignore`.
- **📊 Multi-Format Exports**: Outputs include SQLite, JSON, Graphviz DOT, and Markdown.
- **🏢 Multi-Project Support**: Can index multiple root directories into a single unified graph.
- **🌌 3D Visualizer**: Interactive force-directed graph for human architectural exploration.

## 📖 Usage Guide

### 1. Indexing (The Default Mode)
Run the mapper in your project root to create the `.codemap/` index. This is what you should tell your AI agent to read.
```bash
3d [path]
```
- **Outputs**:
    - `.codemap/README.md`: Human/AI project overview.
    - `.codemap/summary.json`: Full machine-readable knowledge graph.
    - `.codemap/tree.txt`: Plain directory structure.
    - `.codemap/graph.dot`: Graphviz file for dependency visualization.

### 2. Watch Mode (Live Updates)
Keep the index perfectly synchronized with your changes.
```bash
3d --watch
```

### 3. 3D Visualization
Open an interactive 3D map of your project in the browser.
```bash
3d 3d [path]
```
The server will start at `http://localhost:3000`.

### 4. Multi-Project Mapping
Index several folders into one unified graph.
```bash
3d path/to/frontend path/to/backend
```

### 5. Self-Documentation (Stub Generation)
To automatically add purpose comments to files:
```bash
3d --stubs
```

The mapper automatically respects your `.gitignore`. You can add additional patterns by creating a `.codemapignore` file in your project root.
